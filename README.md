# SAP MM Procure-to-Pay (P2P) Project

## 📌 Overview
This project demonstrates the complete Procure-to-Pay (P2P) cycle in SAP MM, covering the full purchasing process from Purchase Requisition to Vendor Payment.

## 🔄 Process Flow
PR → RFQ → PO → GR → Invoice → Payment

## ⚙️ Modules Used
- SAP MM (Materials Management)
- SAP FI (Financial Accounting)

## 🧾 Key Transactions
- ME51N – Purchase Requisition  
- ME41 / ME47 – Request for Quotation  
- ME21N – Purchase Order  
- MIGO – Goods Receipt  
- MIRO – Invoice Verification  
- F110 – Automatic Payment  

## 🖼️ Screenshots Included
- Purchase Order Creation (ME21N)  
- Goods Receipt (MIGO)  
- Invoice Verification (MIRO)  
- Payment Run (F110)  

## 📂 Project Files
- SAP_P2P_Project.pdf  
- images/ (screenshots + flow diagram)

## 🎯 Objective
To demonstrate how SAP automates procurement processes using a structured workflow and integrates with financial accounting through real-time postings.

## ✨ Key Features
- End-to-end P2P cycle  
- 3-way matching (PO–GR–Invoice)  
- Real-time FI integration  
- Automated vendor payments
